Extract cubesÂ¶

Now extract a cube. First, you need to create an appropriate .ini file by modifying sample.ini in the code directory. With the example file names given above and with your modified .ini file in the current working directory, you would run:

extractcube /path/to/CRSA00000000.fits modified.ini

The arguments are simply 1. The raw file(s) to extract into cubes 2. The configuration file The extracted cubes will be written to the current working directory. The first HDU is simply the header with some basic information, the second HDU is the cube, the third HDU is the inverse variance on the cube, and the fourth HDU has no data but saves the original header on HDU0 of the raw reads.
